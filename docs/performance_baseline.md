## 8. 关键观察

### 性能特征
- **GPU 为主的计算模式**: 训练 CPU 利用率仅 26%，大部分计算卸载到 Metal GPU
- **内存极其充裕**: 128GB 统一内存中仅使用 ~3GB（2.3% 利用率），有巨大提升空间
- **VRAM 报告仅 21.5MB**: MLX 使用统一内存，实际 Metal buffer 占用远大于此数字
- **评估比训练更 CPU 密集**: 评估 CPU 利用率 54% vs 训练 26%，因为评估是纯推理（无 GPU 训练操作）

### 瓶颈分析
- 当前配置下 M3 Max 40 GPU 核心远未饱和（564K 参数太小）
- 回放缓冲区在 cycle 10 (~32s) 即填满 50K，之后持续淘汰旧数据
- 每 cycle ~3.4s: 自对弈 ~2.5s + 训练 50 步 ~0.9s
- 纯策略网络 (MCTS=0) 推理极快，自对弈瓶颈在游戏逻辑而非网络推理

### 已知限制
- **Metal buffer limit**: macOS 限制单进程 499,000 个 Metal buffer。BatchNorm 在 train 模式下每次前向传播分配新 buffer，长时间评估会触发限制。已通过 `model.eval()` + 定期 `mx.clear_cache()` 修复。
- **无 MCTS**: 当前使用纯策略网络 (MCTS_SIMULATIONS=0)，搜索能力有限
