### 3.3 PL 关键观察

1. **C301-400 的 PL 回弹现象**：PL 在 C200 触达 3.815 后，C300-400 区间出现 **反向增长**（slope +0.00078/cycle），均值从 3.668 上升到 3.774（+0.106）。这是一个极其重要的现象：
   - 与 v2 的 C201-250 停滞带（slope ≈ 0）不同，8×96 出现了 **实质性的 PL 回弹**
   - 回弹期间 WR 反而从 0% 升到了 **3.3%-12.5%**——说明 PL 回弹并非退化，而是 **策略重组**
   - 推测：模型在此阶段正在从简单模式（直线进攻）向复杂模式（多方向防守/攻击）转型，转型期间 policy 的确定性暂时降低（PL 升），但战术有效性在提升（WR 升）

2. **PL 下降呈"台阶式"而非单调下降**：
   - 台阶 1 (C1-200): 5.41 → 3.82 (Δ-1.59, 快速下降)
   - 平台 1 (C200-400): 在 3.67-3.77 之间震荡约 200 cycles (~23h)
   - 台阶 2 (C400-700): 3.77 → 3.52 (Δ-0.25, 缓慢下降)
   - 平台 2 (C700-900): 在 3.45-3.48 之间震荡约 200 cycles (~12h)
   - 台阶 3 (C900-1120): 3.39 → 3.31 (Δ-0.08, 极缓慢下降)
   - 每一级台阶的落差递减：1.59 → 0.25 → 0.08，收敛的边际回报急剧下降

3. **PL 波动幅度**：
   - C1-100: PL range = 1.22 (5.42-4.20)
   - C501-600: PL range = 0.45 (3.37-3.81)
   - C1001-1120: PL range = 0.49 (3.07-3.56)
   
   后期 PL 波动幅度约 0.45-0.49，相对于均值约 14% 的 CV。这种高波动说明 **模型在不同 cycle 之间的表现差异仍然很大**——策略稳定性有待提升。

4. **PL 未完全 Plateau 但接近极限**：C1001-1120 的 slope 为 -0.00045/cycle（远小于早期的 -0.01207），且 PL 最低值 3.071（C828）在后续 300 cycle 中未能被超越。**8×96 的 PL 下限可能在 3.0-3.1 附近**。

## 8. Learning Rate 分析

### 8.1 LR 配置

本次实验 **全程使用 LR=0.0005**，未手动降低。

对比 v2：84db (12h, LR=0.0005) + 073e (2h, LR=0.0003) = 两阶段 LR。

### 8.2 LR 不衰减的影响

| 对比维度 | v2 (后 2h, LR=0.0003) | v3 (后 5h, LR=0.0005) |
|---------|----------------------|----------------------|
| PL slope (最后 200c) | -0.00099/cycle | -0.00045/cycle |
| PL 波动 (std) | ~0.08 | ~0.12 |
| WR 波动 (std) | 5.1% | 5.9% |

**观察**：

1. **PL 波动更大**：LR=0.0005 全程导致后期 PL 的 std 更高（0.12 vs v2 后期的估计 0.08）。PL 在 3.07-3.56 之间大幅震荡。

2. **PL slope 更平**：尽管 LR 更高应该带来更大的更新幅度，但后期 slope 反而更平（-0.00045 vs v2 的 -0.00099）。这可能因为 **LR 过高导致在局部最优间来回振荡**，表面上看 slope 平缓，实际上是振荡抵消了进步。

3. **v2 的 LR 衰减决策被再次验证**：v2 报告建议的 LR schedule（C400 后降到 0.0003，C700 后降到 0.0002）在本次实验中未被采用，**导致后半段训练效率可能低于最优**。

### 8.3 假设性分析：如果本次实验使用 LR 衰减

根据 v2 的 PL slope 数据推算：

- 如果在 C400 (14h) 降 LR 到 0.0003，PL slope 可能翻倍（从 -0.00078 到 -0.0015）
- C400-1120 的 720 cycles 中可能额外获得 PL Δ ≈ -0.36
- 最终 PL 可能达到 ~3.0 而非 3.23
- PL 3.0 对应的 WR 可能在 60-65%（而非当前的 50%）

**结论：LR 未衰减是本次实验的一个遗憾。如果 v2 的建议被采纳，预计能在相同时间内获得更高的 WR。**

## 10. 训练动态与健康指标

### 10.1 C200-400 策略重组期的深度分析

这是本次实验最独特的现象：

| 阶段 | PL 变化 | VL 变化 | WR 变化 | 解读 |
|------|---------|---------|---------|------|

1. 模型正在 **放弃一些错误的确定性**（PL 升 = entropy 增 = 策略更不确定）
2. 但同时学到了 **更有效的战术**（WR 升 = 实战能力更强）
3. 这本质上是 **模型从"自信但错误"转向"谨慎但正确"的过程**

## 12. Verdict — 总结性分析

### 12.1 核心结论

**8×96 (1,454,163 参数) + 39h 连续训练在 M4 Pro 上达到了 vs C-L1 的 50.0% WR (200g final)**，最后 10 次 probe 均值 54.8%。这是 v15 系列首次 **稳定超越 C-L1 minimax 对手**。

### 12.2 v15 的终极发现

1. **模型尺寸有两面性**：8×96 (1.45M) 比 8×64 (713K) 最终 WR 高 5.5 个百分点，但训练时间多 2.8 倍。参数翻倍带来的 WR 提升远不是线性的——**这是严重的"大模型效率陷阱"**。

2. **8×96 的真正优势在于天花板更高**：8×64 在 14h 后 WR 增速已明显放缓（PL→WR 边际递减严重），而 8×96 在 39h 后仍有上升势头（最后两次 probe 均为 63.3%）。**如果给 8×96 更多时间 + LR 衰减，预计能冲击 WR 70%+。**

3. **PL 回弹是正常的学习阶段**（C300-400）：PL 上升不等于退化。"策略重组期" 是模型从简单模式转向复杂模式的必经之路。未来训练中不应因为短期 PL 回弹而恐慌。

4. **方向性能力的再平衡**：8×96 成功修复了 v2 中 8×64 的主对角线弱势（28.9% → 57.1%），但付出的代价是部分水平/垂直开局退步。**策略空间的"此消彼长"** 暗示 self-play 数据的分布偏移可能需要修正（如 D4 对称增强或开局均衡采样）。

5. **LR 衰减的缺失是可量化的遗憾**：全程 LR=0.0005 导致后期 PL 波动大（±0.25）、收敛效率低。如果采用 v2 建议的 LR schedule，预计最终 WR 可再提升 5-10 个百分点。

6. **训练密度 s/p 在不同模型间不可直接比较**：8×96 在 s/p=38 达到 WR 50%，而 8×64 在 s/p=38 仅约 WR 20%。**每个参数"学到的知识量"不同——更大模型的参数效率（WR per s/p）更高，但时间效率更低。**

7. **120-game probe 的统计噪声仍然较大**：95% CI ≈ ±9%。建议未来使用 200-game probe 以获得更精确的 WR 估计（95% CI ≈ ±7%）。

### 12.3 训练时间投入产出比

| 从→到 | 所需时间 | WR 增幅 | 每小时 WR 增幅 |
|-------|---------|---------|---------------|
| 0% → 10% | 13.6h | +10% | 0.74%/h |
| 10% → 30% | 8.5h | +20% | 2.35%/h |
| 30% → 50% | 4.5h | +20% | **4.44%/h** |
| 50% → 60% | 10.3h | +10% | 0.97%/h |

**WR 30-50% 区间是训练效率最高的甜区**（4.44%/h）。0-10% 需要漫长的 warmup，而 50%+ 后进入边际递减。

### 12.4 下一步建议

1. **LR 衰减**：在 WR 40%+ 后将 LR 从 0.0005 降到 0.0003，WR 55%+ 后降到 0.0002
2. **Probe games**：增加到 200 games 以提高 WR 测量精度
3. **方向性平衡**：考虑增加开局多样性或 D4 对称增强以修复垂直方向弱势
4. **Eval level 提升**：当前 WR vs L1 已稳定超过 50%，可以开始对 L2 进行试探性评估
5. **时间预算**：预计达到 WR 70% vs L1 需要额外 20-30h（含 LR 衰减优化）

v2 最终 PL=3.39，v3 最终 PL=3.40——**几乎完全一样**。但 WR 从 44.5% 提高到 50.0%。

这说明 **PL 数值不能跨模型比较**。对于 8×64 (713K 参数)，PL=3.39 意味着每个参数承载了更高密度的策略信息（s/p=53）。对于 8×96 (1.45M 参数)，PL=3.40 意味着每个参数承载了更低密度的信息（s/p=38），但 **参数之间的协同效应** 产生了更好的战术决策。

**类比**：两台电脑存储了相同 "总量" 的知识（PL 相同），但一台内存更大（8×96）——它可以同时调用更多知识来做决策，即使每块知识的密度更低。

#### 发现 2：更大模型在更低 s/p 达到更高 WR

| 模型 | 达到 WR 35% 的 s/p | 达到 WR 45% 的 s/p |
|------|-------------------|--------------------|
| 8×64 | ~46 (C430, 8h) | ~53 (C784, 14h) |
| 8×96 | ~22 (C640, 22h) | ~25 (C730, 25h) |

8×96 用 **不到一半的 s/p** 达到了相同甚至更高的 WR。这意味着 **每单位训练密度产生的战术学习更高效**——但 **每单位时间** 的效率更低。

这是一个经典的 scaling law 表现：更大模型的 **样本效率** 更高（需要更少的数据就能学到同样的东西），但 **计算效率** 更低（每个样本需要更多的计算）。

#### 发现 3：方向性能力的戏剧性重组

v2 的核心弱点是主对角线 D\\（28.9% WR）。v3 不仅修复了它（57.1%），还将反对角线 D/ 提升到了 66.7%。但代价是：
- 先手优势退步：57% → 42.9% (-14pp)
- 部分垂直/水平开局退步（Opening 6: 42% → 25%, Opening 7: 58% → 42%）

**这揭示了一个深层问题**：当前的 self-play 训练方式下，策略改进是 **零和的**——模型的注意力和学习资源有限，学会了对角线就"遗忘"了部分其他方向的战术。要打破这个限制，可能需要：
- D4 对称数据增强（将一个方向的战术知识映射到所有 8 个等价方向）
- 开局均衡采样（确保每个开局方向在 self-play 中有等概率出现）
- 更大的 buffer（保留更多样化的训练数据）

### 13.3 v2 预测的准确性回顾

v2 报告（§14）对 8×96 三阶段训练做了详细的辩证分析和时间预测。让我们看看预测的准确性：

| v2 预测 | 实际结果 | 评估 |
|---------|---------|------|
| S1 (0→40%): ~28h | C730 (~25h) 达到 45% | ⚠️ 接近但乐观 |
| S1 cycle time: ~44s | 实际 ~120-130s | ❌ 严重低估 |
| S1 s/p/h: 1.44 | 实际 0.98 | ❌ 高估 47% |
| "SPC=100 存在隐患" | 实际使用 SPC=200 | ✅ 建议被采纳 |
| "全程 PG=64" | 实际使用 PG=64 | ✅ 建议被采纳 |
| "全程 MCTS=400" | 实际使用 MCTS=400 | ✅ 建议被采纳 |
| "LR 衰减必要" | 实际未衰减 | ⚠️ 建议正确但未被采纳 |
| "WR 80% 可能不可达" | 39h 达到 50% | ⚠️ 观点方向正确 |
| "方向性偏差可能是天花板" | 偏差格局变化但仍存在 | ✅ 观点正确 |

**v2 的超参建议大部分正确**（SPC=200、PG=64、MCTS=400），但 **时间预测严重低估了 8×96 的 cycle 时间**——预测 44s vs 实际 120s 差 2.7 倍，这是因为 v2 基于理论计算而非实际测量。

### 13.4 终极反省

#### 反省 1：我们低估了"时间成本"这个维度

整个 v15 系列的一条暗线是：**最终 WR 的提升越来越多地依赖于纯粹的训练时间**，而不是超参调优或架构改进。

| 改进 | WR 提升 | 成本 | 效率 |
|------|---------|------|------|
| 修复 SPC (30→200) | 0% → 0% | 1h 代码修改 | 前提条件 |
| 缩小模型 (8×128→4×64) | 0% → 3% | 2h 训练 | 高 |
| 找到甜点 (4×64→8×64) + PG 64 | 3% → 44.5% | 14h 训练 | 高 |
| 换更大模型 (8×64→8×96) + 39h | 44.5% → 50.0% | 39h 训练 | **极低** |

从 44.5% 到 50.0% 的 5.5 个百分点，花了 **39 小时**——平均每个百分点需要 7 小时。而从 3% 到 44.5% 的 41.5 个百分点只用了 14 小时——平均每个百分点只需要 20 分钟。

**效率下降了 21 倍。**

#### 反省 2：LR schedule 是"免费的午餐"

v2 明确建议了 LR 衰减策略（C400 后 → 0.0003, C700 后 → 0.0002），但 v3 未采纳。基于 v2 中 LR=0.0003 使 PL slope 翻倍的实证，这可能在零额外成本下获得 5-10pp 的 WR 提升。

**在未来的长时间训练中，LR cosine annealing 或 step decay 应该作为默认配置**，而非可选优化。

#### 反省 3："容量-密度两难"的实证解答

v15-update §17 提出了 "容量-密度两难"：

```
小模型: 密度高但容量不够
大模型: 容量够但密度不足
```

v3 的 39h 实验给出了部分解答：

- **8×64 (713K) 的天花板确实存在**：14h, s/p=53 后 WR 增速放缓
- **8×96 (1.45M) 确实能突破这个天花板**：WR 从 44.5% 到 50%+
- **但突破的代价是不成比例的**：参数 ×2, 时间 ×2.8, WR 仅 +5.5pp

这个两难的本质是一个 **Pareto 前沿问题**：

```
时间效率优先 → 8×64, 14h, WR 44.5% (3.2%/h)
绝对性能优先 → 8×96, 39h, WR 50.0% (1.3%/h)
```

**两者没有绝对的优劣，取决于你的目标函数。** 如果要在 24h 内获得最高 WR，选 8×64；如果要冲击 WR 60%+，选 8×96 + LR 衰减。

#### 反省 4：v2 的 §14.5 推荐方案回顾

v2 推荐了一个 "两阶段统一方案"：
- Stage A (LR=5e-4, target WR 55%): 预估 ~58h
- Stage B (LR=2e-4, target WR 80%): 预估 ~53h
- 总计 ~111h (4.6 天)

实际 v3 结果表明：
- 仅 Stage A 用了 39h 达到了 WR 50%（接近但未达 55%）
- v2 预测的 58h 对 Stage A 来说偏乐观（实际 cycle time 被低估）
- 如果要继续训练到 WR 80%，即使 LR 衰减，总时间可能需要 **80-120h (3.3-5 天)**

### 13.5 一句话总结

> **v15-v3 实验（8×96, 39h, 50% WR）证实了更大模型可以突破小模型的性能天花板，但代价是训练效率急剧下降（21倍）。LR 未衰减是主要遗憾。核心洞见：同一 PL 值在不同模型尺寸下对应不同 WR（PL 不可跨模型比较），更大模型的样本效率更高但计算效率更低。这是一个经典的 scaling law 场景，下一步突破需要的不是更多时间，而是 LR schedule、数据增强等结构性改进。**
