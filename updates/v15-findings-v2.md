### 3.3 PL 关键观察

1. **C201-250 的临时停滞带**：PL 在 3.98 附近出现 ~50 cycle 的平台（slope ≈ 0），然后在 C300 后恢复下降。这与 §14 中 4×64 和 8×96 在 PL 4.0-4.2 区间的 plateau 吻合——**但 8×64 成功突破了这个平台**。这证明：
   - 4×64 (416K) 的 plateau 确实是容量限制
   - 8×64 (713K) 有足够容量穿越这个区间
   - **PL 4.0 附近存在一个"策略转型带"**——模型需要从简单模式（水平/垂直直线）转向更复杂的模式（对角线、多方向封堵）

2. **PL 波动幅度随训练减小**：
   - C1-100: std=0.327, CV=6.6%
   - C201-400: std=0.107, CV=2.7%
   - C601-784: std=0.129, CV=3.7%
   
   后期 CV 轻微回升（2.7%→3.7%），可能因为模型策略更复杂后对 buffer 中不同 opening 的表现差异更大。

3. **PL 未 Plateau**：即使在 C751-800，PL slope 仍为 -0.00061/cycle，且最低 PL 触达 3.285 (C780)。**8×64 的容量天花板尚未到达**。

## 4. Value Loss 分析

### 4.1 VL 阶段性下降

| 阶段 | 平均 VL | 最小 VL | 最大 VL | Slope |
|------|---------|---------|---------|-------|
| C1-100 | 0.4336 | 0.2259 | 0.7884 | -0.007670 |
| C101-200 | 0.3197 | 0.1889 | 0.4325 | -0.000681 |
| C201-300 | 0.2265 | 0.1394 | 0.3605 | -0.000828 |
| C301-400 | 0.1553 | 0.0412 | 0.3143 | -0.001166 |
| C401-500 | 0.1081 | 0.0444 | 0.2067 | -0.000005 |
| C501-600 | 0.0951 | 0.0226 | 0.1583 | -0.000099 |
| C601-700 | 0.0949 | 0.0297 | 0.1952 | -0.000064 |
| C701-800 | 0.0920 | 0.0268 | 0.1693 | -0.000109 |

### 4.2 VL 关键观察

1. **VL 在 C500 后进入平台 (~0.09)**：从 C500 开始，VL 均值稳定在 0.09-0.10，不再显著下降。这说明 value head 的胜负预测能力已接近当前数据分布下的天花板。

2. **VL 的绝对值解读**：VL=0.089 意味着 MSE 损失在 0.089 左右。由于 value target 是 ±1（胜/负），这相当于预测误差约 ±0.30。模型大约知道 "谁在赢"，但精度有限。

3. **VL 波动依然很大**：即使在后期，VL 在 0.03-0.19 之间震荡（6 倍范围），说明不同 buffer batch 中的胜负分布差异很大。这是 self-play 数据本身的特性——有的 cycle 中模型赢得多（VL 低），有的 cycle 中输得多（VL 高）。

4. **VL vs PL 的独立性**：PL 从 C500 的 3.66 降到 C784 的 3.39（持续改善），但 VL 在同期几乎不变（0.095→0.089）。**Policy 能力的提升没有伴随 Value 预测的同步改善**——这可能因为 value head 的容量有限（8×64 中 value head 只是一个小 FC 层）。

### 5.3 WR vs PL 关系

| PL 范围 | Probes | 平均 WR | 最高 WR | 最低 WR |
|---------|--------|---------|---------|---------|
| 5.0-5.5 | 3 | 0.0% | 0.0% | 0.0% |

| 4.0-4.5 | 11 | 0.7% | 1.9% | 0.0% |

| 3.6-3.8 | 8 | 25.9% | 39.4% | 18.8% |
| 3.4-3.6 | 14 | 34.2% | 44.4% | 21.2% |
| 3.2-3.4 | 4 | 33.8% | 38.1% | 29.4% |

**PL-WR 关系是非线性的**：

- PL > 4.0: WR ≈ 0%（模型太弱，即使偶尔赢也是运气）
- PL 3.8-4.0: WR 跳跃到 ~8%（**关键阈值区间**）
- PL 3.6-3.8: WR ≈ 26%（线性增长）
- PL 3.4-3.6: WR ≈ 34%（增长放缓）
- PL 3.2-3.4: WR ≈ 34%（**边际收益递减**）

**发现**：PL 从 3.6 降到 3.4（Δ0.2）仅带来 WR 从 26% 到 34% 的提升（+8%），而 PL 从 4.0 降到 3.8（同样 Δ0.2）带来了从 0.7% 到 8.1% 的提升（+7.4%）。从 PL 3.4 开始，PL 的边际收益明显递减——**进一步提升 WR 可能需要 PL 大幅下降，或者需要战术质量的结构性改善**（而非单纯的 PL 数值下降）。

### 5.4 WR 稳定性

### 6.2 方向性发现

1. **主对角线 (D\\) 仍然是最薄弱的方向**：28.9% WR vs 反对角线的 55.6%——差距 26.7 个百分点。这与 §13 (c4b44746) 中发现的 "对角线封堵仅 5%" 问题一脉相承。

2. **反对角线 D/ 意外地强**：55.6% 甚至超过水平方向。这可能因为反对角线 [(7,7)+(5,9)] 和 [(7,7)+(9,5)] 的开局构成了一种模型学会利用的特殊模式。

3. **水平方向进步最快**：C667→C784 之间 WR 提升 15.8 个百分点（36.8%→52.6%），说明 resume 阶段（LR=0.0003）的 2 小时主要强化了水平方向的战术。

4. **主对角线几乎无进步**：仅 +2.6 个百分点（26.3%→28.9%）。这再次确认 D4 对称性增强无法将水平/垂直的战术知识映射到 45°对角线——模型需要独立学习对角线封堵，而当前训练量不足以充分覆盖这个方向。

### 6.4 极端开局的解读

- **Opening 15 (83% WR)**：(7,8)→(8,7) 构成了一个非对称的开局，模型在此开局下几乎压倒性获胜。这可能是模型发现了一种特定的战术序列。
- **Opening 1 (14% WR) 和 Opening 3 (14% WR)**：对手占据中心后走对角邻位，模型极度被动。这是主对角线弱势的直接体现。
- **Opening 10 vs Opening 11 (50% vs 25%)**：两者都是主对角线远邻，但 (5,5) 和 (9,9) 的 WR 差距巨大。这暗示模型的对角线学习存在不对称性——可能只学会了某一个方向的对角线防守。

## 7. Buffer 与 Steps-per-Cycle 分析

### 7.1 Buffer 饱和

| Run | Buffer 满时间 | Cycle | 说明 |
|-----|-------------|-------|------|
| 84db (12h) | 25.5 min | C13 | PG=64 → 64×~60 pos/game ≈ 3840 pos/cycle |
| 073e (resume) | 34.3 min | C707 | 从空 buffer 重新充填 |

Buffer 在 C13 (~25min) 即饱和——这比 §14 中 PG=16 的 C83-C105 (32-61min) 更快，完全符合 PG=64 带来的 4 倍数据生成速度。

### 7.2 SPC 实际值

- 配置 SPC = 200
- Buffer = 50,000, Batch = 256
- 有效 SPC = floor(50,000 / 256) = **195**
- 整个训练过程中 SPC 始终被 buffer/batch 比值 cap 在 195

**195 步 = buffer 中全部样本遍历一次（1 epoch per cycle）**——这是理想状态。模型在每个 cycle 中恰好将 buffer 中的数据完整训练一遍，既不过度拟合（多遍），也不遗漏数据（不到一遍）。

### 7.3 Resume 阶段的 SPC 损失

Resume 启动后 buffer 从 0 重新充填，导致前 ~40 个 cycle 的 SPC 远低于 195：

| Cycle (global) | Buffer Size | SPC |
|----------------|-------------|-----|
| C668 | 1,217 | 4 |
| C675 | 10,096 | 39 |
| C685 | 22,250 | 86 |
| C695 | 33,908 | 136 |
| C707 | 50,000+ | 195 |

**估算损失**：在 C668-C707 的 40 个 cycle 中，累计损失约 **3,704 步**（相当于 19 个完整 cycle 的训练量）。这占 resume 总 18,876 步的 20%——**resume 机制的效率代价不可忽略**。

### 7.4 SPC=200 是否需要调整

**结论：当前 SPC=200 (实际 195) 是合理的。**

证据：
1. **1 epoch/cycle** 是 RL self-play 的经典比例——每次生成新数据后用完一遍即可
2. 如果增加 SPC 到 400（需要 buffer=102,400），则每 cycle 训练两遍——增加对 buffer 旧数据的过拟合风险
3. 如果减少 SPC 到 100，则每 cycle 只用一半数据——浪费
4. PG=64 带来的 4 倍数据刷新率意味着每条新数据在 buffer 中平均存活 52 个 cycle，被训练 52×195/50000 ≈ 203 次——这个比例在合理范围内

### 7.5 Buffer 大小是否需要调整

**结论：当前 buffer=50,000 是合理的。**

- Buffer 翻转率 = 总生成位置 / buffer 容量 = 3,015,385 / 50,000 = **60.3 次**
- 每 cycle 刷新率 = 64 games × ~60 pos ÷ 50,000 = **7.7%**
- 数据新鲜度高，没有出现 "stale data" 相关的训练退化迹象

如果增大 buffer（如 100K），会让旧数据停留更久（来自更弱版本的模型），可能拖慢收敛。

## 8. Learning Rate 分析

### 8.1 两个 run 的 LR 配置

| Run | LR | 阶段 |
|-----|-----|------|
| 84db3b78 (12h) | **0.0005** | 全程固定 |
| 073ee01b (2h resume) | **0.0003** | 手动降低 |

### 8.2 LR 降低的效果

| 指标 | 84db 最后 100 cycles (C567-667) | 073e 全程 (C668-784) |
|------|--------------------------------|---------------------|
| LR | 0.0005 | 0.0003 |
| 平均 PL | 3.601 | 3.455 |
| PL Slope | -0.00056/cycle | **-0.00099/cycle** |
| 平均 WR | 35.4% | 36.6% |
| WR 最高 | 41.9% | **44.4%** |

**LR 从 0.0005 降到 0.0003 后**：
1. PL 均值降低了 0.146（3.601→3.455）—— 显著改善
2. PL slope 从 -0.00056 加速到 -0.00099 —— **收敛速度几乎翻倍**
3. WR 仅微幅提升 (+1.2%)，但 WR 在此区间本身增长放缓（边际递减）

### 8.3 LR 分析判断

**降低 LR 到 0.0003 是正确的决策**，原因：

1. **PL 收敛加速**：LR 越低，梯度更新越小，但方向更准——在后期（模型已经学到了主要特征，需要微调战术细节时），小 LR 避免了在局部最优间来回振荡。

2. **PL 波动减小**：resume 阶段的 PL std 预计更低（尽管 buffer 重建阶段有干扰），说明参数更新更稳定。

3. **但进一步降低需要谨慎**：LR=0.0003 在 2h (117 cycles) 内 PL 仅从 ~3.55 降到 ~3.39（Δ0.16），如果 LR 再降到 0.0001，每 cycle 的进步将进一步缩小，可能在有限时间内无法获得足够的 PL 改善。

### 8.4 LR Schedule 建议

基于当前数据，84db+073e 的合理 LR 时间线是：
- C1-400 (0-7h): LR=0.0005（快速探索）
- C400-700 (7-12h): LR=0.0003（精细化）
- C700+ (12h+): LR=0.0002 或 0.0001（微调阶段）

当前代码使用固定 LR，不支持 schedule。如果未来实现 LR cosine annealing 或 step decay，可能会获得更好的收敛效率。

### 9.1 Entropy 数值

| 指标 | 值 | 说明 |
|------|-----|------|
| 均匀 entropy | 5.42 nats | 225 个位置等概率 |
| 最终 entropy | **3.13 nats** | |
| Entropy 下降 | **2.29 nats (42.2%)** | |
| 有效候选数 | e^3.13 ≈ **22.9 个位置** | |
| Focus | **23%** | 最高概率的 move 占总概率的 23% |

### 9.2 与前期 runs 对比

| Run | 模型 | PL | Entropy | Focus | WR |
|-----|------|-----|---------|-------|-----|
| c2ae880e | 4×64 | 4.18 | 3.06 | 29% | 3.0% |
| 83e2edea | 8×96 | 3.98 | 3.06 | 29% | 0.5% |
| **84+073e** | **8×64** | **3.39** | **3.13** | **23%** | **44.5%** |

### 9.3 Entropy 悖论

**8×64 在 WR 44.5% 时的 entropy (3.13) 反而比 4×64 在 WR 3% 时的 entropy (3.06) 更高，focus (23%) 比之前 (29%) 更低。**

这看似矛盾，实际反映了一个重要的现象：

1. **更强的模型不一定更集中**：弱模型可能只学会了一两种简单模式（如总是试图在中心附近连子），因此 entropy 低、focus 高。但这种 "假集中" 不代表强度。

2. **8×64 的策略更 diverse**：更高的 entropy 意味着模型在不同局面下做出不同的选择，而不是简单地重复同一种模式。这种 "有意义的多样性" 正是它能赢更多局的原因——它能根据对手的走法灵活调整战术。

3. **Focus 23% 是合理的**：最佳走法只占 23% 的概率，说明模型仍然在 "犹豫"——但这可能也包含了 MCTS 探索带来的分散效应。在 self-play 中（使用 temperature > 0），policy 本身就不应过度集中。

### 9.4 Entropy Gap

Entropy gap = 理论最优 entropy - 当前 entropy = policy_loss - entropy ≈ 3.39 - 3.13 = **0.26 nats**

这个 gap 衡量了 policy 的 "过度自信错误"——模型给出的概率分布与实际最优分布之间的交叉熵差。0.26 nats 的 gap 远小于早期的 4+ nats，说明模型的概率校准有了显著改善。

> **注**：TUI 截图显示 entropy gap 约 4.14-4.18 nats（对于 §14 的旧模型），而当前 gap 已缩小到 ~0.26 nats——这是训练质量的直接证据。

## 10. Signal 信号密度分析

### 10.1 Signal 指标定义

从 `train.py` 源码分析，TUI 中显示的 **Signal** 指标是 **corrective_ratio**——即每个训练 batch 中 "纠正性样本" 的比例。

**纠正性样本** 定义为：
- 来自对手视角的位置 (`source == "opponent"`)
- 或来自最终输了的对局 (`value < 0.0`)

这些样本代表了模型犯错的地方——模型需要从这些错误中学习。Signal 越高，意味着模型面对更多的 "教训"。

### 10.2 Signal 的采样机制

```python
FOCUSED_SAMPLE_RATIO = 0.25  # 25% 的 batch 保证是纠正性样本
```

采样策略：
1. 从 buffer 中标记所有 "纠正性" 样本（对手视角 + 输了的对局）
2. 强制抽取至少 25% 的 batch (64/256) 为纠正性样本
3. 剩余 75% 按 recency-weighted priority 随机采样
4. 最终的 corrective_ratio 可能高于 25%（因为随机部分也可能包含纠正性样本）

### 10.3 当前 Signal 值

TUI 截图显示：**Signal = 76%, Avg = 76%**

这意味着每个 batch 中有 76% 的样本是纠正性样本——远高于 FOCUSED_SAMPLE_RATIO 的 25% 保底。

### 10.4 Signal 76% 的含义

**Signal 76% = buffer 中 76% 的样本来自 "对手视角" 或 "输了的对局"**

这非常高，说明：
1. **模型在 self-play 中仍然输多赢少**：WR vs C-L1 是 44.5%，但 self-play 是模型 vs 自身——所以一半的对局中模型作为一方输了（self-play 本身是 50/50）。**但 76% 包含了对手视角的样本**，所以实际上：
   - 每局有 ~50% 的手是 "输家" 的手 → value < 0
   - 对手视角的手也被标记为 corrective
   - 因此 76% 是合理的——大约 50%（输家手）+ 部分额外的对手视角手

2. **Signal 作为训练信号的质量指标**：高 Signal 不一定是坏事——它意味着模型有大量的 "学习材料"。在训练早期（WR ~0%），Signal 可能接近 100%（几乎所有样本都是 "教训"）。随着模型变强，Signal 应该逐渐下降（因为赢的对局增多）。

### 10.5 Signal 对训练的影响分析

**Signal 是我们过去分析的盲点**——之前的分析集中在 PL、VL、WR、Entropy 上，从未评估过训练 batch 的信号质量。

Signal 76% 的影响：
1. **正面**：模型持续从错误中学习，不会出现 "已经太强以至于没有错误可学" 的问题
2. **正面**：25% 的 focused sampling 保底确保即使 buffer 中纠正性样本比例变化，模型也能获得足够的 "教训"
3. **潜在风险**：如果 Signal 过高（接近 100%），可能意味着 buffer 中几乎没有 "正面案例"（赢了的手），模型可能变得过度保守——只知道什么不该做，不知道什么该做
4. **当前 76% 处于健康区间**：有足够的纠正信号，同时还有 24% 的正面示例

### 10.6 Signal 趋势推断

从 TUI 截图的 sparkline 图表看，Signal 在整个训练过程中维持在 70-80% 范围，没有明显的趋势变化。这符合 self-play 的本质——无论模型多强，self-play 总是产生 ~50% 的输家视角数据。

**建议**：在未来的训练中监控 Signal 指标的趋势。如果 Signal 突然下降到 < 50%（意味着 buffer 中大部分是 "赢家" 数据），可能需要调整 FOCUSED_SAMPLE_RATIO 或重新审视 self-play 数据的标注逻辑。

### 10.7 Length 相关 Signal

TUI 还显示：
- **Boost**: 99% — 几乎所有样本都获得了 length-based 加权提升
- **Damp**: 0% — 几乎没有样本被抑制
- **Length multiplier**: x1.17

这说明当前的 length priority 机制几乎是全面 boost、无 damp——所有对局都被认为 "值得加权学习"。Boost 99% 意味着 length priority 机制实质上退化为一个均匀的 1.17× 放大器，失去了差异化调节的作用。

> **观察**：如果所有样本都被 boost，等价于没有 boost。FAST_WIN_LEN 和 SLOW_WIN_LEN 的阈值可能需要根据当前对局长度分布重新校准，以恢复差异化的样本优先级。

## 13. Verdict — 总结性分析

### 13.1 核心结论

**8×64 (713K 参数) + 14h 训练 是 v15 阶段的最佳方案**，实现了对 C-L1 的 44.5% WR——从零到接近均势的飞跃。

### 13.2 v15 的关键发现

1. **模型尺寸选择是最关键的决策**：从 8×128 (2.49M, 0% WR) → 4×64 (416K, 3% WR) → 8×64 (713K, 44.5% WR)，三次模型调整直接决定了训练成败。过大的模型（8×128、8×96）在合理时间内无法获得足够的训练密度；过小的模型（4×64）在训练密度充足后撞上容量天花板。**713K 是当前框架的甜点。**

2. **训练时间的回报是超线性的**：前 6 小时 WR 仅 ~7%，6-12 小时 WR 从 7% 跃升到 35.5%，12-14 小时再到 44.5%。存在一个 "积累到爆发" 的过程——模型需要先学会基础模式（水平/垂直防堵），然后才能开始赢棋。

3. **PL 与 WR 的关系有一个关键转折区**：PL 4.0→3.8 是 WR 从 0% 到 >5% 的关键区间。PL 3.8→3.6 是 WR 加速增长的区间。PL 3.4 以下 WR 增长开始减缓。

4. **方向性能力不均衡是核心短板**：反对角线 55.6% WR vs 主对角线 28.9% WR 的差距（26.7 个百分点）说明模型的战术学习存在严重的方向偏差。D4 对称增强无法解决 45° 方向的映射问题。

5. **LR 衰减在后期有效**：从 0.0005 降到 0.0003 使 PL 收敛速度从 -0.00056 加速到 -0.00099/cycle（几乎翻倍），但 WR 提升有限——说明 WR 在 35-45% 区间的提升需要的不仅仅是 PL 下降。

6. **Signal (corrective ratio) 76% 是新发现的关键指标**：高 Signal 意味着模型持续面对丰富的学习材料。Signal 76% 配合 FOCUSED_SAMPLE_RATIO=25% 的保底机制，确保了训练信号的质量。但 Length Boost 几乎退化为均匀放大器（99% boost, 0% damp），失去了差异化效果。

7. **Buffer 与 SPC 的配合是正确的**：50K buffer + 256 batch → 195 effective SPC ≈ 1 epoch/cycle，配合 PG=64 的 7.7% 刷新率，数据新鲜度充足。

8. **Resume 机制有约 20% 的效率损失**：buffer 从零重建消耗 ~40 个 cycle。连续长时间训练优于 resume 拼接。

### 13.3 v15 阶段数据 Verdict

> **v15 阶段通过 6 个训练 run 和 4 次关键超参调整，将对 C-L1 的 WR 从 0% 提升到 44.5%。核心路径是：修复 eval 性能 → 发现对手强度失效 → 诊断 spc 瓶颈 → 诊断模型尺寸瓶颈 → 找到 8×64 甜点 → 充分训练时间。当前模型在水平和反对角线方向已具备竞争力（50-56% WR），但主对角线方向 (29%) 和特定对手开局后的应对 (14%) 仍然薄弱。PL 仍在缓慢下降（未 plateau），WR 仍有提升空间，但 PL→WR 的边际收益已明显递减——下一步突破可能需要超参微调之外的结构性改进。**
